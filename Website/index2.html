Overview

We have made a Taxi application similar to Uber.

It runs on Android verion 4.1 or newer.

Pre-requisites

For this project we have used:

  • Android Studio
  • FireBase
  • Geofire
  • Glide
  • Place Autocomplete
  • Google Maps API
  • Flaticon for the icons

The application can run on the phone and you will need: Android version 4.1 or newer, internet connection, active location and a well-functioning batery.

Setup and Build plan

First we created our Drivy project on Android Studio.

Secondly we created a database using Firebase so we can store the driver and customer data the app receives and works with.

For the customer side, there is a request button , which will retrieve the user's location and notify the closest driver. The customer can choose between DrivyX and DrivyBlack, which offers increased comfort.

There is also a search bar which lets the customer select the destination. This was implemented using the Places API from google.

The settings button will open a new activity, where the user can insert his/her data in dedicated text edits. An image view also gives them the possibility to add a profile picture. On the bottom of the screen there are 2 buttons: the confirm button will save everything in the database, while the back button will take you back to customer_map_activity.

The logout button will sign out the user and take him back to the main activity.

The driver_map_activity is similar to the customer one: it has a settings button, which will open a new activity, where the driver can input personal information.

When a request is made by a customer, the driver will see its lcation and personal data. It will also show the route to the customer, for an easier navigation.

Meanwhile, the customer is able to see how far the driver is and also his contact number.

The 'logout' button will sign out the driver and take him back to the main page, same as for the customer.
